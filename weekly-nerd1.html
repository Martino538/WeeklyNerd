Home Nerd 1 - Kilian Valkhof

JavaScript? Overbodig!

Datum: 07-02-2024

Spreker: Kilian Valkhof

Korte samenvatting

De eerste spreker is Kilian Valkhof. Hij maakt een browser voor ontwikkelaars genaamd Polypane. Deze browser focust zich op webontwikkeling en is voor ontwikkelaars een enorm goede toevoeging. Voor andere zaken naast webontwikkeling is het een minder goede browser vertelde Kilian, dus daar moet je het niet voor willen gebruiken.

Ook vertelde hij over een regel genaamd Rule of power. Hiermee bedoeld hij dat het een goed uitgangspunt is om de programmeertaal te kiezen die het minst kan, maar wel precies doet wat jij wilt.

Hiernaast verdiept hij zich ook veel in CSS. Zo komen er steeds meer technieken die je voorheen met Javascript moest oplossen, maar nu ook kunt maken d.m.v. CSS. Er is veel minder code vereist om een functionaliteit te laten werken. Hiernaast kan het soms zijn dat je grote JavaScript bestanden moet inladen waardoor de website trager wordt. Het is dan veel beter om dit met CSS op te kunnen lossen.

Aantekeningen

Hierdoor zijn mijn aantekingen weergegeven:

  • Soms is het leuk om een toggle knop te hebben, maar dit wordt van nature niet zomaar ondersteund. Al snel ben je geneigd om je framework te raadplegen om dit probleem te verhelpen. Dit is niet altijd nodig. Het is namelijk mogelijk om een checkbox om te bouwen tot een toggle knop. Voeg 'appearance: none' toe aan je checkbox. Hierdoor kan je de knop.
  • Input elementen hebben van nature geen before & after. Met appearance: none maak je dit wel mogelijk in CSS.
  • Input:focus-visible is ook nieuw. Hierbij gaat het erom dat je interactie kunnen hebben met een element, maar dan zonder je muis te gebruiken. Een mooi voorbeeld hiervan is je toetsenbord. Door een outline toe te voegen kan je zien waar je bent.
  • Als je als input type 'color' gebruikt krijg je een color picker.

Reflectie

Deze lezing vond ik erg interessant! Door de manier waarop hij alles vertelde begreep alles. Ik vind het bijzonder om te zien wat je hier als webdeveloper allemaal mee kan. Hiernaast verdiept hij zich ook veel in CSS. Zo komen er steeds meer technieken die je voorheen met Javascript moest oplossen, maar nu ook kunt maken d.m.v. CSS. Er is veel minder code vereist om een functionaliteit te laten werken. De technieken die mij in het bijzonder zijn bijgebleven zijn de Accordion & Dialog. Dit bied mogelijkheden om zoveel sneller en makkelijker een functionaliteit te bouwen zonder al te veel JavaScript te gebruiken. Deze technieken zoals input types, dialogs en details/summary ga ik zeker meenemen naar de toekomst.
